## 1. Установка

Установка делается **один раз**.

### Mac

1. Скачай файл установщика:
   👉 https://github.com/AnvarBakiyev/marketing-dronor/raw/main/install.sh

2. Открой Terminal (Finder → Программы → Утилиты → Terminal)

3. Введи команду и нажми Enter:
   ```
   bash ~/Downloads/install.sh
   ```

4. Следуй подсказкам (~10 мин)

### Windows

1. Скачай файл установщика:
   👉 https://github.com/AnvarBakiyev/marketing-dronor/raw/main/install.bat

2. Найди скачанный файл `install.bat`

3. Правая кнопка мыши → **Запуск от имени администратора**

4. Следуй подсказкам (~15 мин)

### Что устанавливается автоматически

- Python, Git, PostgreSQL 16
- Все Python-пакеты + браузер Chromium
- База данных со всеми схемами
- Ярлык на рабочем столе

Отдельно установи **GoLogin** (без него отправка не работает):
https://gologin.com/download

## 2. Ежедневный запуск

1. Запусти **GoLogin** (иконка на рабочем столе)
2. Дважды кликни ярлык «Запустить Marketing Dronor»
3. Откроется браузер: `http://localhost:8899`
4. Войди с паролем, который указал при установке

> ⚠️ GoLogin должен быть запущен **ДО** начала отправки сообщений.

## 3. Command Center

Панель управления системой в браузере. Работает только локально, интернет не нужен.

| Вкладка | Что делать |
|---------|------------|
| Operator Queue | Проверять и одобрять сообщения |
| Analytics | Статистика отправок и ответов |
| Data Pipeline | Запускать модули системы |
| Conversations | История переписки |
| Accounts | Управление Twitter-аккаунтами |
| Proxies | Управление прокси |
| Admin | Настройки системы |

---

## 4. Как работает система

```
[M1] Сбор 200K Twitter-профилей
[M2] Анализ: кому стоит написать (founder / vc / dev / exec)
[M3] Прогрев 56 аккаунтов
[M4] AI генерирует персонализированные DM
[M5] <- ТВОЯ РАБОТА: проверить и одобрить
[M5] Отправка DM через GoLogin + прокси
[M6] Мониторинг ответов + follow-up
[M7] Поиск твитов для контекста
```

Твоя роль — шаг M5: проверить сообщения, одобрить хорошие.

---

## 5. Работа оператора

### Утро (15–20 мин)

**Шаг 1.** Data Pipeline → **M4 Run** (AI генерирует новые сообщения)

**Шаг 2.** Operator Queue — проверь каждое сообщение:
- Approve — хорошее, отправить
- Edit — подправить текст, потом одобрить
- Reject — не отправлять

Правило: одобряй только то, что звучит естественно. Шаблонные — отклоняй.

**Шаг 3.** Data Pipeline → **M5 Run** (отправка всех одобренных)

### Вечер (5 мин)

**M6 Run** — обработка ответов, follow-up
**Analytics** — проверь цифры за день

## 6. Вкладки

### Operator Queue

Главная рабочая вкладка. Фильтры по статусу, категории, аккаунту.

Сообщения `approved` отправляются **только после M5 Run**. Автоматически не уходят.

### Data Pipeline

| Кнопка | Что делает |
|--------|-----------|
| M1 Run | Собрать новые профили Twitter |
| M2 Run | Проанализировать профили |
| M4 Run | Сгенерировать DM |
| M5 Run | Отправить одобренные сообщения |
| M6 Run | Обработать ответы, follow-up |
| M7 Run | Найти твиты для контекста |

### Accounts

Список 56 Twitter-аккаунтов. У каждого:
- **Status** — active / warmup / paused
- **GoLogin Profile ID** — ID профиля в GoLogin (заполняется вручную)
- **Daily limit** — макс DM в день
- **Warmup phase** — этап прогрева (1–4)

Не меняй лимиты без согласования.

### Proxies

- Зелёная = прокси работает
- Красная = мёртвый прокси
- **Check / Check All** — проверить
- **+ IMPORT** — добавить список

### Analytics

Статистика: Sent DM, Replies, Conversion rate — по категориям и дням.

## 7. GoLogin

GoLogin нужен, чтобы каждый аккаунт выглядел как реальный пользователь с отдельным IP.

### Привязка профиля к аккаунту

1. GoLogin → **+ Create profile** → название `mkt_account_01`
2. Скопируй **Profile ID** (длинный код в настройках профиля)
3. Command Center → **Accounts** → нужный аккаунт → вставь в **GoLogin Profile ID**

1 профиль GoLogin = 1 Twitter-аккаунт. Никогда не используй один профиль для двух аккаунтов.

### Прокси в GoLogin

1. Открой профиль → вкладка **Proxy**
2. Формат: `host:port:user:password`
3. **Check proxy** — должна появиться страна

---

## 8. Прокси

Proxies → **+ IMPORT** → формат:
```
user:password@host:port
```

Если прокси красный — нажми **Check**. Если не помогло — сообщи администратору.

## 9. FAQ

**Где взять API-ключи?**
- TwitterAPI.io: https://twitterapi.io (ключ начинается с `ta_`)
- Anthropic: https://console.anthropic.com (начинается с `sk-ant-`)

**Command Center не открывается:**
1. Запусти ярлык «Запустить Marketing Dronor»
2. Открой вручную: `http://localhost:8899`
3. Не помогло — перезапусти ярлык

**GoLogin выдаёт ошибку при отправке:**
1. GoLogin Desktop запущен? (иконка в трее)
2. Перезапусти GoLogin
3. Попробуй M5 Run снова

**Сообщения не отправляются — чек-лист:**
- [ ] GoLogin запущен
- [ ] У аккаунтов заполнен GoLogin Profile ID
- [ ] Прокси зелёные
- [ ] Есть сообщения `approved` в Operator Queue
- [ ] Нажал M5 Run

**Как изменить API-ключи?**

Файл `.env` в папке `MarketingDronor`:
- Mac: `~/MarketingDronor/.env`
- Windows: `%USERPROFILE%\MarketingDronor\.env`

**DRY_RUN=true — что это?**

Тестовый режим — сообщения **не отправляются** реально. Для боевого: `DRY_RUN=false`.
